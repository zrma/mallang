# Mallang v1.1.0 Release Notes

## Status

Mallang 1.1.0 is the first backward-compatible minor GitHub binary release in
the stable v1 line:

```text
https://github.com/zrma/mallang/releases/tag/v1.1.0
```

Crates.io publication remains disabled.

## Highlights

- `std/fs.forEachLine[C,S]` processes UTF-8 files one line at a time with
  borrowed context, mutable borrowed state, a one-based line number, and a
  synchronous visitor.
- Runtime storage is bounded by the longest line plus a fixed input buffer,
  rather than total file size.
- LF is omitted, CR and embedded NUL are preserved, a terminal LF creates no
  phantom line, and invalid UTF-8 is a recoverable `errors.Error`.
- Native file handles and borrowed returns remain outside the source language;
  no syntax, keyword, ownership rule, or existing v1 observable behavior changes.
- The release gate installs published v1.0.0, upgrades to v1.1.0, rolls back,
  and re-upgrades while preserving representative existing-v1 output and
  validating the new streaming API.
- macOS arm64 and Linux x86_64 run the complete compiler, strict C, sanitizer,
  release artifact, clean-install, and compatibility acceptance.

## Install

```sh
curl -fsSLO https://github.com/zrma/mallang/releases/download/v1.1.0/install.sh
chmod +x install.sh
./install.sh --version 1.1.0
```

`clang` is required for Mallang native build, run and test commands.

## Verification

```sh
scripts/check-v1x-acceptance.sh
scripts/check-v1x-upgrade.sh
scripts/verify-v0-rc.sh
```

The signed tag points at the synchronized compatible API specification,
conformance map, package version, release notes, acceptance record, and
supported-platform artifact contract.
