# Mallang v1.2.0 Release Notes

## Status

Mallang 1.2.0 is the compatible default self-hosted compiler release:

```text
https://github.com/zrma/mallang/releases/tag/v1.2.0
```

Crates.io publication remains disabled.

## Highlights

- The public `mlg` command now uses the fixed-point Mallang compiler core by
  default. The internal engine is installed as the sibling `mlgc` executable.
- The complete public `check`, `ir`, `build`, `run`, `fmt` and project `test`
  workflows execute Mallang-owned lexer, parser, package graph, semantic and
  ownership checks, typed IR, formatter, test runner and C backend logic.
- The tracked Rust workspace and lockfile remain the reproducible Stage0 seed,
  differential oracle and explicit `--compiler stage0` recovery path. A
  self-hosted failure never triggers silent fallback.
- A clean checkout builds Stage0, Stage1 and byte-identical Stage2 without an
  opaque prebuilt compiler. The complete differential, fixed-point, strict C,
  allocation-accounting and sanitizer gates preserve the frozen v1 contract.
- Release archives contain the verified `mlg`/`mlgc` pair. The installer also
  recognizes the published v1.0.0 legacy single-driver archive so rollback
  removes any stale core and a current re-upgrade restores the pair.
- macOS arm64 and Linux x86_64 pass default-compiler, fixed-point, release
  archive, checksum, clean-install, v1.0.0 rollback and current re-upgrade
  acceptance.

## Compatibility

This is a backward-compatible minor tooling and distribution release. It adds no
source syntax or standard-library API and preserves the Mallang v1 language,
ownership, diagnostic, CLI and supported-platform contracts.

## Install

```sh
curl -fsSLO https://github.com/zrma/mallang/releases/download/v1.2.0/install.sh
chmod +x install.sh
./install.sh --version 1.2.0
```

`clang` is required for Mallang native build, run and test commands.
